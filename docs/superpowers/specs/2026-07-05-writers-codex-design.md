# Writer's Codex — 分层记忆 + 审查系统设计

**日期**: 2026-07-05
**状态**: 设计已确认，待实现

## 概述

为小说角色模拟器构建 Writer's Codex 系统，让 Agent 在 1M 上下文窗口中获得完整的写作素材（前置约束），并在每章写完后通过独立审查 Agent 确保输出质量（后置审查）。

## 背景

调研 330+ 来源后发现行业共识：

| 来源 | 关键发现 |
|------|---------|
| SCORE 论文 | 混合检索 + 动态状态追踪，一致性提升 23.6%，幻觉减少 41.8% |
| DOME 论文 (NAACL 2025) | 时序知识图谱存储故事，矛盾减少 87.61% |
| TRACE 论文 (CHI 2026) | 作者偏好"先写后审"(Write-Audit-Correct) |
| 中文写作工具社区 | Truth File 模式、流水线多 Agent、伏笔作为一等公民 |
| SAGA / InkOS / NovelClaw | 知识图谱 + 结构化正典 + 多 Agent 编排 |

核心洞察：当前项目的 **1M 上下文窗口**可以取代传统 RAG 的检索功能，直接全量注入素材。

## 架构

### Codex 七段结构

写前组装，注入 system prompt + 首个 user message：

| 段 | 内容 | 预估 Token |
|----|------|-----------|
| 1. 风格包 | 原著文风分析、代表性片段 × 10-15、风格指纹 | ~20K |
| 2. 角色卷宗 | 完整档案、关键语录、角色弧线、当前状态快照 | ~50K |
| 3. 世界观百科 | 力量体系、社会结构、势力分布、关键地点 | ~30K |
| 4. 前文摘要层 | 全文章节摘要、最近 3 章完整前文、当前章节大纲 | ~50K |
| 5. 伏笔账本 | 已埋未收/已回收伏笔、类型、建议回收窗口 | ~10K |
| 6. 灵感库 | 写作技巧参考、同类桥段参考、作者笔记 | ~20K |
| 7. 当前任务 | 本轮场景目标、节奏要求、情节指令 | ~5K |

**总计 ~185K tokens**，1M 窗口利用率约 19%。

### 审查维度（写后）

| 审查 Agent | 检查什么 | 对照来源 |
|-----------|---------|---------|
| 角色一致性审查 | 说话风格漂移、行为是否符合性格、关系动态 | 段 2 |
| 连贯性审查 | 因果链断裂、物体凭空出现、已死角色说话 | 段 4 + 段 2 |
| 伏笔追踪审查 | 新伏笔登记、已有伏笔是否到回收窗口 | 段 5 |
| 风格一致性审查 | 句式/用词/对话比是否符合指纹、AI 味检测 | 段 1 |
| 世界观审查 | 力量体系规则打破、地点描述矛盾 | 段 3 |
| 节奏检查 | 冲突强度、节拍数是否符合计划 | 段 7 |

### 工作流

```
Writer 输出草稿
  → 5 个审查 Agent 并行检查
  → 合并审查报告
  → 自动修正 minor 问题
  → 标记 major/critical 给作者
  → 更新 Codex（状态/摘要/伏笔）
```

### 生命周期

**初始化（一次性）：**
```
原始小说 → Extractors（已有: CharacterExtractor, StoryExtractor, TimelineExtractor）
  → 填充角色卷宗、世界观百科、前文摘要、风格包、伏笔账本
  → 补齐: 角色语录提取、风格指纹计算
```

**每章循环：**
```
1. 取最新状态快照 → 组装 Codex（全 7 段）
2. Writer Agent 创作
3. 5 审查 Agent 并行检查
4. 自动修正 minor / 标记 major+ / 人类确认
5. 更新 Codex: 角色状态、章节摘要、伏笔登记、前文滚动
6. 下一章 ← 循环
```

## 实现范围

### Phase 1: Codex 构建器
- 新建 `src/core/codex/builder.ts` — 组装 7 段 Codex
- 新建 `src/core/codex/updater.ts` — 写完后的状态更新
- 提取角色语录（从原文搜索角色对话，选代表性片段）
- 计算风格指纹（句长分布、对话/叙述比、常用句式/词汇）
- 补充更多文风范例片段（3-5 → 10-15）

### Phase 2: 审查 Agent 系统
- 新建 `src/core/reviewer/` — 扩展现有 3 个审查 Agent 到 6 个
- 新建审查编排器 — 并行运行 + 结果合并 + 自动修正 minor
- 审查结果接入 Codex updater

### Phase 3: 集成到模拟引擎
- Writer Agent 注入 Codex（替代当前的简单 context）
- 审查 Agent 在每轮/每章后自动运行
- 管理后台可查看/编辑 Codex 各段内容

## 关键设计决策

1. **1M 上下文替代 RAG** — 不需要向量数据库，不需要检索。全量注入 Codex，简单可靠
2. **先约束后审查** — 结合 TRACE 论文的 Write-Audit-Correct + 中文社区的 Truth File 模式
3. **Codex 是活的** — 每一章写完自动更新，避免手动维护负担
4. **minor 自动修，major 人类定** — 不引入全自动循环（防止越改越差）
